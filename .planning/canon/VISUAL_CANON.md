# Visual Canon — CRM RAI

**Источник:** workspace.pro-gripil.online/consulting/crm (RAI Platform)
**Статус:** ОБЯЗАТЕЛЕН для всех UI-фаз (redesign + новые экраны)
**Применение:** Все шаблоны, компоненты, стили должны соответствовать этому канону.

---

## 1. Типографика (Geist Canon)

**Шрифты:** `Geist Sans` (UI) / `Geist Mono` (технические данные). Другие шрифты ЗАПРЕЩЕНЫ.

| Класс | Назначение | Tailwind |
|---|---|---|
| `font-medium` | Заголовки, кнопки, метрики | `font-medium` |
| `font-normal` | Основной текст | `font-normal` |
| `font-bold` | ЗАПРЕЩЕН | — |
| `font-semibold` | ЗАПРЕЩЕН | — |
| Mono | ID, логи, финансы, счета, БИК, ИНН | `font-mono` |

**Принцип:** Акцент создаётся пространством, а не жирностью.

**Подключение Geist:**
```html
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/geist@1/dist/font/geist-sans.css">
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/geist@1/dist/font/geist-mono.css">
```
Или через npm: `npm install geist` (если перейдём на сборку).

Tailwind config:
```js
fontFamily: {
  sans: ['Geist Sans', 'system-ui', 'sans-serif'],
  mono: ['Geist Mono', 'monospace'],
}
```

## 3. Семантические статусы (Risk Triage)

| Уровень | Категория | Цвет | Tailwind | UX поведение |
|---|---|---|---|---|
| **R4** | Critical | Red-600 | `text-red-600` / `bg-red-600` | Hard-escalation, блокировка действий |
| **R3** | High | Amber-600 | `text-amber-600` | Красный баннер, требование Quorum |
| **R2** | Moderate | Amber-400 | `text-amber-400` | Жёлтый индикатор, запись в Deviation |
| **R1** | Info | Blue-500 | `text-blue-500` | Синий маркер, AI-агент |
| **Success** | Verified | Emerald-500 | `text-emerald-500` | Данные закреплены, транзакция исполнена |

**Применение в CRM RAI:**
- Стадия `lost` → R4 (Red-600)
- Просроченный таск → R3 (Amber-600)
- Стадия `0` (сырой лид) → R1 (Blue-500)
- Стадия `7` (оплачено) → Success (Emerald-500)
- Ворота воронки не пройдены → R2 (Amber-400)

## 4. Режим View / Edit (B-Pattern)

### View Mode (по умолчанию)
- Поля = Label + Value, без рамок и input-стилей
- Пустые значения → `—` (цвет Muted)
- Никаких интерактивных селектов/пикеров

### Edit Mode
- Появляются input-границы `border-black/10`
- Обязательные поля → красная звёздочка `*`
- Валидация отображается ПОД полем, не ломая layout

**Применение:** Карточка лида — по умолчанию View Mode. Кнопка «Редактировать» переключает на Edit Mode. Сохранение → обратно в View.

## 5. Иерархия CTA (Call to Action)

На одной странице — ровно ОДНО действие Primary.

| Уровень | Стиль | Tailwind | Назначение |
|---|---|---|---|
| **Primary** | Black/Indigo | `bg-[#030213] text-white` или `bg-indigo-600 text-white` | Глобальное сохранение, финализация |
| **Secondary** | Outline/White | `bg-white border border-black/10` | «Добавить», «Отмена», «Редактировать» |
| **Ghost/Text** | Text only | `text-[#717182] hover:text-[#030213]` | Удаление, второстепенные опции |

## 6. Tabular Scalability (C-Pattern)

- Все списки (счета, контакты, лиды, документы) → табличный вид
- Карточки для списков ЗАПРЕЩЕНЫ (кроме канбан-колонок)
- Scalability: 20+ записей без деградации
- **Drawers (SidePanelForm):** создание/редактирование элементов — в боковой панели, сохраняет контекст
- **Empty States:** иконка + текст + кнопка Add в центре пустой таблицы

**Исключение:** Канбан-доска — карточки внутри колонок (это board, не list).

## 7. Контекстные подсказки вместо плашек

- Длинные информационные плашки с очевидным текстом ЗАПРЕЩЕНЫ
- **Паттерн:** кнопка-подсказка `i` или `?` рядом с заголовком
- Поведение: открывается по наведению/фокусу
- Язык: без жаргона, англицизмов, процессного формализма
- **Исключение:** постоянные баннеры только для: ошибка, риск, блокировка, эскалация, несохранённые изменения, обязательное следующее действие
- **Запрет:** не дублировать описание страницы длинным абзацем под заголовком

## 8. AI Dock и оверлэй-окна (перспективно)

Для будущих AI-функций CRM:
- **AI Dock слева:** `header → signals → history → composer` (единая оболочка)
- **Signals:** компактный встроенный блок, без тяжёлых карточек
- **Правая зона:** рабочая поверхность продукта
- **Окна агента:** overlay поверх правой зоны
- **Window modes:** `inline` (краткий сигнал) / `panel` (рабочий режим) / `takeover` (глубокий режим)

## Миграция текущего UI

Текущий CRM RAI (после Фазы 3) использует:
- Tailwind CSS (CDN) — ✅ совместим
- Шрифт по умолчанию (system-ui) — ❌ нужен Geist
- `bg-gray-100` фон — ❌ нужен `bg-slate-50`
- `font-bold` / `font-semibold` в шаблонах — ❌ запрещены
- Карточки-счётчики на дашборде — ✅ допустимы (не список)
- Канбан-карточки — ✅ допустимы (board)
- Списки в `<table>` — ✅ уже табличные
- HTMX-формы inline — ⚠️ нужно перевести на Drawer-паттерн

**Redesign-фаза** применит канон ко всем экранам после функционального завершения Фазы 3.

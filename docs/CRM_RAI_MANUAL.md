## 2. Роли пользователей

| Роль | Возможности |
|---|---|
| **Менеджер** | Видит свои лиды на канбане, ведёт карточки, звонит, создаёт задачи, генерирует документы |

## 3. Дашборд

Дашборд (`/`) — главная страница. Содержимое зависит от роли.

### Для менеджера

- **Всего моих лидов** — количество лидов, назначенных на вас
- **Звонков сегодня** — количество записей в журнале контактов за сегодня
- **Просроченных** — задачи с прошедшей датой (подсвечены красным)
- **На сегодня** — активные задачи
- **Моя воронка** — распределение ваших лидов по стадиям
- **Просроченные задачи** — блок с красной подсветкой
- **Задачи на сегодня** — блок с предстоящими задачами

### Для супервайзера / администратора

- **Всего лидов** — общее количество в системе
- **Уровни A / B / C** — распределение по приоритету
- **Воронка по стадиям** — все лиды по 9 стадиям
- **По регионам** — таблица с количеством лидов в каждом регионе

## 4. Канбан — воронка продаж

**Адрес:** `/kanban`

Канбан — главный рабочий экран менеджера. Лиды распределены по 9 колонкам-стадиям.

### Стадии воронки

| Код | Стадия | Описание |
|---|---|---|
| 0 | Сырые лиды | Данные от супервайзера, не проверены |
| 1 | В работе | Менеджер звонит, ищет ЛПР |
| 2 | Квалифицирован | Рапс подтверждён, объём и ЛПР известны |
| 3 | КП отправлено | Коммерческое предложение отправлено |
| 4 | Переговоры | Обсуждение условий |
| 5 | Договор | Договор подписан |
| 6 | Счёт выставлен | Счёт на оплату |
| 7 | Оплачено | Сделка закрыта |
| lost | Потерян | Отказ |

### Перетаскивание карточек

Карточки перетаскиваются между колонками мышью (drag-and-drop через SortableJS).

### Правила перехода (ворота)

Не все переходы разрешены. Система проверяет условия:

| Переход | Условие |
|---|---|
| 0 → 1 | Лиду должен быть назначен менеджер |
| 1 → 2 | Среди контактов должен быть отмечен хотя бы один ЛПР + рапс подтверждён |
| Любая → lost | Всегда разрешено (нужна причина потери) |
| Остальные | Свободный переход |

Если условие не выполнено — карточка вернётся в исходную колонку, появится сообщение с причиной.

### Фильтры

Над канбаном — панель фильтров:

- **Регион** — показать лидов только выбранного региона
- **Уровень** — фильтр по A / B / C
- **Приоритет** — фильтр по очереди (1, 2, 3)
- **Все лиды / Мои лиды** — доступно супервайзеру/админу

### Карточка лида на канбане

Каждая карточка содержит:
- Название компании (ссылка на карточку лида)
- Бейдж уровня (A — зелёный, B — жёлтый, C — серый)
- Приоритет (П1, П2, П3)
- Регион

## 5. Карточка лида

**Адрес:** `/leads/{id}`

Карточка лида — полная информация о контрагенте. Открытие — клик по названию компании на канбане.

### Вкладки

Карточка содержит 7 вкладок:

#### 5.1. Информация

Реквизиты компании и данные по рапсу. По умолчанию — режим просмотра (View Mode):

- **ИНН** — в моноширинном шрифте
- **Руководитель**
- **Сайт**
- **Уровень** — A / B / C
- **Приоритет** — 1 / 2 / 3
- **Рапс подтверждён** — Да / Нет
- **Объём рапса** — тонн / гектаров
- **Тайминг уборки** — когда урожай
- **Комментарий для CRM** — общий текст
- **Что сделано** — история действий
- **Что нужно сделать** — план

Кнопка **«Редактировать»** переключает в режим редактирования (Edit Mode) — появляются поля ввода. Кнопка **«Сохранить»** сохраняет изменения и возвращает в режим просмотра.

**Назначение менеджера** (только супервайзер/админ) — отдельная форма в нижней части вкладки.

#### 5.2. Контакты

Список контактных лиц компании:
- Имя
- Должность
- Телефон (моноширинный)
- Email
- Флаг **ЛПР** (лицо, принимающее решение) — чекбокс

Кнопка **«+ Добавить»** открывает боковую панель (drawer) с формой добавления контакта.

Чекбокс ЛПР переключается одним кликом — обновление происходит без перезагрузки.

**Важно:** хотя бы один контакт должен быть отмечен как ЛПР для перехода из стадии «В работе» в «Квалифицирован».

#### 5.3. Журнал

Хронологический журнал контактов с компанией (звонки, встречи, письма):

- Дата и время
- Тип (звонок, email, визит)
- Результат — текстовое описание
- Исход — классификация (не дозвонился, занято, перезвонить, договорились, отказ, отправлено КП)
- Дата следующего действия — если указана, автоматически создаётся задача «Перезвонить: {название компании}»

Кнопка **«+ Добавить»** открывает drawer с формой.

#### 5.4. Комментарии

Свободные заметки менеджера. Новые комментарии добавляются вверху списка. Форма — в drawer.

#### 5.5. Задачи

Задачи, привязанные к данному лиду:
- Название
- Дата дедлайна
- Статус (Ожидает / В работе / Готово / Отменено)

Смена статуса — через выпадающий список без перезагрузки.

#### 5.6. Документы

Сгенерированные документы (КП, договоры, счета):
- Тип документа
- Номер и сумма
- Статус (Черновик / Отправлен / Просмотрен / Принят / Отклонён)
- Ссылки на скачивание (.docx и .pdf)

Кнопка **«Сгенерировать»** — выбор шаблона, заполнение доп. полей, генерация документа с автоподстановкой данных лида.

Если шаблоны не загружены — показывается подсказка со ссылкой на страницу «Шаблоны».

#### 5.7. Сделки

Сделки по данному лиду:
- Название
- Сумма
- Статус (Новая / КП отправлено / Переговоры / Договор / Счёт выставлен / Оплачено / Потеряна)

Создание новой сделки — через drawer.

## 6. Задачи

**Адрес:** `/tasks`

Страница задач текущего пользователя.

### Фильтры

- **Активные** — задачи в статусе «Ожидает» и «В работе»
- **Завершённые** — задачи в статусе «Готово»
- **Все** — полный список

### Отображение

Каждая задача:
- Название
- Дата дедлайна (или «без срока»)
- Ссылка на лид (если привязан)
- Выпадающий список смены статуса

**Просроченные задачи** подсвечиваются жёлтой полосой слева (Amber-600 по канону).

### Создание задач

Задачи создаются автоматически:
- При добавлении записи в журнал с указанием «даты следующего действия»
- Через AI-агента (инструмент `create_task`)

## 7. Документооборот

### Генерация документов

1. Откройте карточку лида → вкладка **«Документы»**
2. Выберите шаблон из выпадающего списка
3. Если шаблон содержит нестандартные поля (помимо автоматических) — появятся поля для ввода
4. Нажмите **«Сгенерировать»**

Система:
- Открывает .docx-шаблон
- Подставляет данные лида (название, ИНН, руководитель, телефон, email и т.д.)
- Заменяет плейсхолдеры вида `{company_name}`, `{inn}`, `{date}`
- Сохраняет .docx и конвертирует в .pdf
- Создаёт запись о документе со статусом «Черновик»

### Автоматические поля (плейсхолдеры)

Эти поля заполняются из данных лида автоматически:

| Плейсхолдер | Источник |
|---|---|
| `{company_name}` | Название компании |
| `{inn}` | ИНН |
| `{district}` | Район |
| `{settlement}` | Населённый пункт |
| `{address}` | Адрес |
| `{head_name}` | Руководитель |
| `{site}` | Сайт |
| `{region}` | Регион |
| `{manager_name}` | ФИО текущего пользователя |
| `{date}` | Текущая дата |
| `{phone}` | Первый контактный телефон |
| `{email}` | Первый контактный email |
| `{rapeseed_volume}` | Объём рапса |
| `{harvest_timing}` | Тайминг уборки |

### Скачивание

Каждый документ имеет две ссылки:
- **.docx** — редактируемый Word-файл
- **.pdf** — PDF для отправки клиенту

### Статусы документов

| Статус | Описание |
|---|---|
| Черновик | Документ создан, не отправлен |
| Отправлен | Документ отправлен клиенту (фиксируется время отправки) |
| Просмотрен | Клиент ознакомился |
| Принят | Клиент согласен |
| Отклонён | Клиент отказался |

## 9. Шаблоны документов

**Адрес:** `/templates`

Управление .docx-шаблонами документов (только супервайзер/админ).

### Загрузка шаблона

1. Нажмите форму загрузки
2. Выберите .docx-файл
3. Укажите название (например «КП ГРИПИЛ»)
4. Выберите тип: КП / Договор / Счёт / Акт
5. Нажмите **«Загрузить»**

Система автоматически:
- Сохраняет файл в `templates_docx/`
- Парсит плейсхолдеры `{field_name}` из текста и таблиц
- Сохраняет список плейсхолдеров в БД

### Создание шаблона в Word

1. Откройте новый документ в Microsoft Word
2. Напишите текст КП/договора/счёта
3. Вместо конкретных значений вставьте плейсхолдеры в фигурных скобках:
   - `{company_name}` — вместо названия компании
   - `{inn}` — вместо ИНН
   - `{amount}` — вместо суммы (это поле появится в форме генерации)
   - `{date}` — вместо даты
4. Сохраните как .docx

**Важно:** плейсхолдеры должны быть в фигурных скобках без пробелов: `{company_name}`, не `{ company_name }`.

### Удаление шаблона

Кнопка **«Удалить»** — мягкое удаление (шаблон помечается неактивным, файл остаётся на диске).

## 10. Аналитика (супервайзер)

**Адрес:** `/reports` (только супервайзер/админ)

### Дашборд аналитики

Три блока:

#### Воронка продаж

Визуальные полосы (CSS-бары) для каждой стадии. Показывает:
- Количество лидов на стадии
- Процент конверсии от предыдущей стадии

Цвета конверсии:
- Красный — менее 25%
- Жёлтый — менее 50%
- Зелёный — 50% и выше

#### Просадки воронки

Таблица переходов между соседними стадиями:
- Откуда → куда
- Было / Стало (количество)
- Конверсия в процентах

Помогает выявить «узкие места» — где лиды «застревают».

#### Топ-10 регионов

Регионы с наибольшим количеством лидов.

### Детальная воронка

**Адрес:** `/reports/funnel`

Полная таблица: регион × стадия. Все 29 регионов.

### KPI менеджеров

**Адрес:** `/reports/managers`

Таблица KPI по каждому пользователю:
- Всего лидов
- Звонков (за период)
- КП отправлено
- Сделок
- Конверсия (лиды на стадии ≥3 / всего лидов)

Фильтр по датам: укажите «С даты» и «По дату» → «Фильтр».

### Экспорт в Excel

**Адрес:** `/reports/export`

Скачивание отчёта в .xlsx с тремя листами:
1. **Воронка** — регион × стадия
2. **KPI менеджеров** — показатели по менеджерам
3. **Просадки** — переходы между стадиями с конверсией

## 11. Чат с AI-агентом

**Адрес:** `/agent`

Чат с AI-ассистентом на базе Hermes. Агент имеет **прямой доступ к данным CRM** через MCP-сервер — может искать лиды, создавать задачи, показывать воронку.

### Возможности агента

Агент может:

| Действие | Пример запроса |
|---|---|
| **Поиск лидов** | «Найди всех клиентов уровня A в Псковской области» |
| **Детали лида** | «Покажи информацию по ООО Тригорское» |
| **Воронка продаж** | «Покажи воронку» |
| **Мои задачи** | «Какие у меня задачи на сегодня?» |
| **Создать задачу** | «Создай задачу: перезвонить ООО Тригорское завтра» |
| **Записать звонок** | «Запиши: позвонил в ООО Скангали, не дозвонился» |
| **Поиск сделок** | «Покажи все сделки в работе» |
| **Создать сделку** | «Создай сделку для ООО Тригорское на 50000» |
| **Список регионов** | «Какие регионы есть в системе?» |
| **Комментарий** | «Добавь комментарий к лиду 1: клиент готов к сотрудничеству» |

### Быстрые подсказки

Под полем ввода — три кнопки:
- **Найти клиентов** — поиск клиентов уровня A
- **Мои задачи** — задачи на сегодня
- **Создать задачу** — шаблон создания

### Очистка истории

Кнопка **«Очистить историю»** — удаляет все сообщения текущего пользователя.

### Важно

- Агент работает с реальными данными CRM
- Поисковые запросы — безопасны (read-only)
- Создание/изменение — агент записывает в БД напрямую через MCP
- Если Hermes недоступен — покажется сообщение об ошибке, история сохранится

## 12. Воронка продаж — стадии и правила

### Полный цикл

```
Сырой лид → В работе → Квалифицирован → КП отправлено → Переговоры → Договор → Счёт → Оплачено
     │          │            │
     │          └────────────┴──→ Потерян (на любой стадии)
     │
     └── Импорт от супервайзера (Hermes/Excel)
```

### Что делать на каждой стадии

| Стадия | Действия менеджера | Что проверить |
|---|---|---|
| **0. Сырый лид** | Найти рабочие контакты, ЛПР | Назначен ли менеджер |
| **1. В работе** | Звонить, уточнять выращивают ли рапс | Найти ЛПР, подтвердить рапс |
| **2. Квалифицирован** | Уточнить объём, тайминг уборки | Объём и тайминг заполнены |
| **3. КП отправлено** | Сгенерировать КП из шаблона, отправить | Документ создан |
| **4. Переговоры** | Обсуждать условия, дозировку, цену | — |
| **5. Договор** | Подготовить договор | — |
| **6. Счёт выставлен** | Выставить счёт | — |
| **7. Оплачено** | Закрыть сделку | Сделка → «Оплачено» |

### Потеря лида

Лид можно перевести в «Потерян» с любой стадии. При потере укажите причину в поле «Причина потери» на вкладке «Информация».

- **Клик по карточке на канбане** → открытие карточки лида
- **Drag-and-drop на канбане** → смена стадии
- **Чекбокс ЛПР в контактах** → один клик, без перезагрузки
- **Дата следующего действия в журнале** → автоматически создаёт задачу-напоминание
